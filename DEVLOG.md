# Developer Log - Personal Finance App

A complete day-by-day version of the process of making this application

---

### Entry 1: Core Logic
- Created a complete terminal based budgeting system, it would intake your income and split it based on how much you chose to put in each category
- Big problem that I noticed there was it was too slow and annoying to manaully enter how much percent you want in each different area

### Entry 2: Improving Logic and Handling
- Completely reworked budgeting system for user, allowed you to just choose from a list multiple services you wanted to use
- Added preset budgeting options that you could choose from
- Added a basic SQL history database, didn't put a visual way to see it though
- Near the end of the session switched into a testing script "dashboard.py" that eventually became the new main script

### Entry 3: Streamlit
- Learned basic streamlit by setting up sliders, graphs, and tables
- Implemented different tabs, one for budgeting, history, and AI insight
- Added buttons that you could use to save budgeting transactions and clear database data
- Tried to setup the chatbot AI but ended up going down a rabbithole and saved it for another day

### Entry 4: AI Chatbot
- Spent most of the session learning how to implement AI Chatbot kept running into new errors each time (felt familiar and similar setup because of experience working with and setting up NemoClaw)
- Almost got AI Chatbot setup but decided to not get stuck for a long time and actually work again on the main part
- Switched around the slider for budgeting and made it a text based input with - + buttons
- One day before vacation so might be a bit of a gap
